# 首頁第一點與第二點的佈置說明

這份說明專門回答你剛剛問的兩件事，也就是首頁優先修改項目中的**第一點：Email、LinkedIn、履歷按鈕**，以及**第二點：Hero 主標與副標**。你可以把這兩項想成首頁最核心的「可聯絡性」與「第一印象」。只要這兩部分先整理好，網站就已經會從示範版變成一個可以正式對外展示的專業首頁。

## 一、第一點怎麼佈置：Email、LinkedIn、履歷按鈕

這一點的核心不是單純把連結換掉，而是要讓訪客在最短時間內知道「要怎麼聯絡你」以及「下一步該點哪裡」。因此，佈置時應該區分成 **Hero 區的主行動按鈕** 與 **Contact 區的正式聯絡資訊** 兩層。

### 1. Hero 區的按鈕應該怎麼放

Hero 區目前有兩個按鈕，這個配置本身是合理的，建議你保留。原因是首頁首屏不應該出現太多按鈕，否則會讓重點變散。最好的做法是保留 **一個主要按鈕** 與 **一個次要按鈕**。

| 按鈕角色 | 建議用途 | 建議文字 | 目的 |
| --- | --- | --- | --- |
| 主要按鈕 | 導向最重要的內容 | `View Selected Work` 或 `Selected Projects` | 引導訪客先看你的代表經歷 |
| 次要按鈕 | 導向履歷或外部專業檔案 | `Download Resume` 或 `View Resume` | 讓對你有興趣的人快速取得正式資料 |

換句話說，**Hero 區不要放 Email 與 LinkedIn**。這些資訊不適合直接塞在首屏按鈕區，因為 Hero 的任務是建立定位與引導瀏覽，而不是一次把所有聯絡方式攤開。Email 與 LinkedIn 比較適合放在頁面底部的 Contact 區，作為明確的轉換出口。

### 2. Contact 區應該怎麼放

Contact 區目前有四個資訊卡，這個方向也建議保留。不過在正式版中，最重要的是讓這四格不要都只是「連結清單」，而是各自扮演不同角色。你可以這樣配置：

| 卡片位置 | 建議內容 | 你應該放什麼 |
| --- | --- | --- |
| 第 1 格 | Email | 你的主要聯絡信箱 |
| 第 2 格 | GitHub | 你的 GitHub repo 或首頁 |
| 第 3 格 | LinkedIn | 你的 LinkedIn 個人頁 |
| 第 4 格 | Resume / Portfolio / Calendly | 你最希望對方採取的下一步 |

這樣做的好處是，訪客不需要自己猜你的網站想引導他去哪裡。對方若想合作，可以點 Email；若想看技術內容，可以點 GitHub；若想看完整職涯背景，可以點 LinkedIn；若想快速行動，可以點履歷或預約連結。

### 3. 第一點的實際佈置建議

如果你要把它做成真正可用的版本，我會建議採取下面的層級：

| 區域 | 應出現的內容 | 應避免的內容 |
| --- | --- | --- |
| Hero 左側按鈕區 | 1 個作品導向按鈕 + 1 個履歷按鈕 | 放太多外部連結 |
| Navbar 右上角 | 1 個簡潔 CTA，例如 `Let’s Connect` | 同時放 Email 與 LinkedIn icon |
| Footer / Contact 區 | Email、GitHub、LinkedIn、Resume/Calendly | 過多次級社群連結 |

也就是說，你整體網站的聯絡層級應該是：**首屏先引導看內容，頁尾再提供正式聯絡與外部檔案**。這會比一開始就塞滿按鈕更成熟，也更接近企業型首頁的節奏。

### 4. 你可以直接替換成什麼內容

你目前最先要換掉的幾個值，建議如下：

| 現在的示意內容 | 正式版建議 |
| --- | --- |
| `otis@example.com` | 你的常用信箱，例如 `yourname@gmail.com` 或你的學術/工作信箱 |
| `https://www.linkedin.com/` | 你的 LinkedIn 個人公開網址 |
| `Download resume` | 指向你的 PDF 履歷、Notion CV 或 Google Drive 連結 |
| `Replace this CTA` | 改成 `View Resume`、`Book a Call`、`See Full Portfolio` 其中一種 |

如果你現在還沒有完整履歷連結，那就先把第四格保留成 `View Resume`，等你整理好 PDF 再接上去。這樣比硬塞一個暫時不完整的外部頁更穩定。

## 二、第二點怎麼佈置：Hero 主標與副標

第二點是整個首頁中最重要的文案區塊。它的任務不是描述你的全部經歷，而是用最短的篇幅說清楚你是什麼樣的人、你處理什麼類型的問題，以及你和一般個人履歷頁有什麼不同。

### 1. Hero 的文案結構應該長什麼樣子

Hero 區建議維持現在的三層結構，也就是 **小標籤、主標、補充副標**。這樣的層級非常適合你目前想做的專業型首頁。

| 文案層級 | 作用 | 建議字數 |
| --- | --- | --- |
| 小標籤（eyebrow） | 定義你網站的類型或角色 | 4 到 8 個字詞 |
| 主標（headline） | 一句說清你的核心定位 | 8 到 16 個英文單字，或 18 到 30 個中文字 |
| 副標（supporting copy） | 補充你處理的領域與價值 | 1 到 2 句 |

這三層中，最重要的是主標。主標不應該寫成履歷摘要，也不應該列出太多技術名詞。它應該先讓人形成清楚印象，例如你是做 **systems integration**、**engineering operations**、**scalable infrastructure** 這類高層級定位，而不是一開始就寫 Rust、Go、Python、C、Wastewater、Research 全部一起出現。

### 2. 主標怎麼寫比較適合你

以你目前網站的方向來看，你最適合的不是過度學術，也不是過度新創，而是偏 **工程整合型專業首頁**。因此主標最好同時滿足三件事：第一，要有系統感；第二，要有工程感；第三，要讓不熟你背景的人也看得懂。

你可以從下面三種方向選一種：

| 方向 | 建議主標寫法 | 適合情境 |
| --- | --- | --- |
| 工程品牌型 | `Systems integration for scalable infrastructure.` | 你想讓網站更像專業品牌首頁 |
| 個人專業型 | `I build reliable systems across software and operations.` | 你想保留比較明確的個人角色 |
| 工程研究整合型 | `Engineering across software, control, and environmental systems.` | 你想保留研究與環工背景 |

如果你問我哪一個最適合你現在這版網站，我會建議你先用第二種，因為它最平衡。它既保留個人網站的人味，又不會像履歷頁那麼直接；同時也和目前首頁整體視覺最一致。

### 3. 副標怎麼寫比較好

副標的作用是補上主標沒說完的細節。最好的寫法不是列技能，而是用一句話說明你如何工作、你跨哪些領域，以及你的價值在哪裡。

你目前的副標方向其實已經不錯，但可以再更貼近你本人，例如：

| 風格 | 建議副標 |
| --- | --- |
| 穩健專業 | `I work across systems integration, process engineering, and data-informed problem solving to turn technical complexity into dependable execution.` |
| 偏個人定位 | `My work connects software thinking, engineering process, and operational reality to make complex systems clearer and more reliable.` |
| 偏研究導向 | `I focus on the intersection of software, control logic, and environmental engineering, with an emphasis on practical system performance.` |

建議你最後只保留 **一段副標**，不要拆成太多句，也不要在 Hero 區放完整人生故事。更長的背景說明應該留到 Profile 區塊。

### 4. Hero 區塊整體應該怎麼排

從版面角度來看，你現在這版 Hero 的結構是好的，建議不要大改。最適合你的排列方式是：左邊為主敘事，右邊為摘要資訊卡。這樣可以讓首頁看起來像企業型首頁，而不是單欄履歷頁。

| 區塊位置 | 建議內容 | 功能 |
| --- | --- | --- |
| 左側大區塊 | 小標籤、主標、副標、兩個 CTA | 建立第一印象與瀏覽起點 |
| 右側第一張卡 | Current focus | 一句話說明你現在關注的專業方向 |
| 右側第二張卡 | Capabilities | 列出 3 個高層級能力詞 |
| 右側第三張卡 | Navigation note 或短版介紹 | 幫助讀者理解網站後面會看到什麼 |

這個排法非常適合你，因為你的背景本來就不是單一職能。你需要的是一個能同時呈現「整體定位」與「摘要模組」的首頁，而不是只有一個大標題加一張照片。

### 5. 第二點最實用的改寫方法

如果你現在想最快速地把 Hero 換成屬於你的版本，我建議你依照下面順序填空：

| 步驟 | 你要回答的問題 | 產出內容 |
| --- | --- | --- |
| 第一步 | 我希望別人先把我理解成什麼角色？ | 主標 |
| 第二步 | 我主要跨哪些領域？ | 副標前半句 |
| 第三步 | 我帶來的價值是什麼？ | 副標後半句 |
| 第四步 | 我想讓訪客先做什麼？ | 主按鈕文字 |
| 第五步 | 我希望對方如何正式聯絡我？ | 次按鈕與 Contact 區配置 |

例如，如果你希望大家先把你理解成「能連結軟體與工程系統的人」，那主標就應該圍繞這件事，而不是先講你會哪些語言。這是首頁與履歷最大的差別。

## 三、我對你目前網站的具體建議

如果要直接給你一版比較適合現在網站的配置，我會建議這樣放：

| 元素 | 建議內容 |
| --- | --- |
| Hero 小標籤 | `Personal Professional Website` 或 `Systems Integration Profile` |
| Hero 主標 | `I build reliable systems across software and operations.` |
| Hero 副標 | `My work connects software thinking, engineering process, and operational reality to make complex systems clearer and more reliable.` |
| Hero 主按鈕 | `View Selected Work` |
| Hero 次按鈕 | `View Resume` |
| Contact 第 1 格 | 真實 Email |
| Contact 第 2 格 | GitHub |
| Contact 第 3 格 | LinkedIn |
| Contact 第 4 格 | Resume 或 Calendly |

這樣配置的好處是，你的首頁會同時具備三種特性：第一，讓人快速理解你不是一般履歷頁；第二，讓專業訪客能很快找到作品與正式資料；第三，讓整體資訊架構維持乾淨，不會因為太多連結而破壞首頁節奏。

## 四、最簡單的執行建議

> 如果你現在只想先把網站變成正式可用版本，請先把 Hero 區保留兩個按鈕，分別改成「看作品」與「看履歷」，再把 Contact 區四格改成 Email、GitHub、LinkedIn、Resume。接著，把 Hero 主標改成一句能總結你角色的定位句，副標則用一到兩句補充你的跨領域能力與價值。這樣就已經是很成熟的首頁配置了。
